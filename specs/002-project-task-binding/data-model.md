# Data Model: Project and Task Binding Foundation

**Feature**: 002-project-task-binding
**Date**: 2026-07-20
**Storage**: Existing per-user SQLx SQLite database, upgraded by migration 0002

## Design rules

1. Cairn-owned IDs are UUIDv7 strings; paths, remote URLs, names, and titles are never
   identity.
2. Feature 001 rows and events are preserved. No migration-generated project, task,
   revision, association, binding, or binding event is allowed.
3. Every Feature 002 mutation appends event(s) and updates projections in one
   `BEGIN IMMEDIATE` transaction.
4. `events.seq` is the global total order. Post-migration events also have explicit
   aggregate type, aggregate ID, and positive per-aggregate sequence.
5. Task revisions, repository associations, and session bindings have no user-facing
   update/delete transition in this feature.
6. Session binding mode and Feature 001 lifecycle state are independent dimensions.

## Identity newtypes

| Type | Representation | Creation |
|---|---|---|
| `ProjectId` | UUID | UUIDv7 |
| `ProjectRepositoryAssociationId` | UUID | UUIDv7 |
| `TaskId` | UUID | UUIDv7 |

## Project

A local project scope. Names are display metadata and may duplicate.

| Field | Type | Rules |
|---|---|---|
| `id` | ProjectId | Stable, authoritative, immutable |
| `name` | string | Required; line endings normalized; surrounding whitespace trimmed; non-empty; not unique |
| `description` | string? | Optional; same limited normalization; internal whitespace preserved |
| `status` | ProjectStatus | `active` or `archived` |
| `created_at` | Timestamp | Immutable RFC 3339 UTC |
| `updated_at` | Timestamp | Changes on metadata/status update |

### Project state transitions

```text
create → active
active --archive--> archived
archived --restore--> active
```

Archived is readable. It rejects new repository associations, tasks, revisions,
bindings, and bound-session starts. Archive/restore does not alter repositories, tasks,
revisions, sessions, bindings, or events.

## ProjectRepositoryAssociation

An immutable exclusive association between a project and a Feature 001 repository.

| Field | Type | Rules |
|---|---|---|
| `id` | ProjectRepositoryAssociationId | Stable UUIDv7 |
| `project_id` | ProjectId | Existing active project at creation |
| `repository_id` | existing repository ID | Existing Feature 001 repository; globally unique in this table |
| `associated_at` | Timestamp | Immutable |
| `event_seq` | integer | Global sequence of `project.repository_associated` |

Rules:

- Same project + repository request returns the existing row and event.
- Repository associated with another project returns
  `REPOSITORY_PROJECT_CONFLICT`; existing state is unchanged.
- The association remains present when the project is archived.
- Removal, transfer, and reassignment are not modeled.
- Every worktree whose `worktrees.repository_id` equals this `repository_id`
  inherits membership. No worktree association rows are created.

## Task

Stable logical work identity permanently owned by one project.

| Field | Type | Rules |
|---|---|---|
| `id` | TaskId | Stable UUIDv7 |
| `project_id` | ProjectId | Immutable permanent owner |
| `title` | string | Required display metadata; normalized at boundaries; non-empty; not unique |
| `latest_revision_number` | positive integer | Starts at 1; increments transactionally |
| `created_at` | Timestamp | Immutable |
| `updated_at` | Timestamp | Advances only when a revision is accepted |

There is no task move, archive, delete, or title-update operation in Feature 002.

## TaskRevision

Immutable task intent.

| Field | Type | Rules |
|---|---|---|
| `id` | TaskRevisionId | Stable UUIDv7 |
| `task_id` | TaskId | Immutable |
| `revision_number` | positive integer | Sequential within task; unique with task ID |
| `parent_revision_id` | TaskRevisionId? | Null for revision 1; otherwise earlier revision of same task |
| `goal_contract_json` | UTF-8 JSON string | Exact canonical GoalContractV1 bytes |
| `goal_contract_schema_version` | integer | 1 for Feature 002 |
| `goal_contract_fingerprint` | 64-char lowercase hex | BLAKE3 of canonical bytes |
| `idempotency_key` | IdempotencyKey | Globally unique persisted retry identity |
| `created_at` | Timestamp | Immutable |

Creation rules:

1. Task creation inserts the task and revision 1 atomically.
2. Revision 1 has number 1 and no parent.
3. A later revision defaults to the immediately previous revision as parent.
4. An explicit parent must belong to the same task and have a lower number.
5. A `BEGIN IMMEDIATE` transaction conditionally increments the task counter and inserts
   under `UNIQUE(task_id, revision_number)`.
6. A matching idempotency retry returns the existing revision.
7. Reuse of an idempotency key with a different operation kind, project/task identity,
   task-create title, parent revision, or canonical contract fails with
   `TASK_REVISION_CONFLICT` for revision operations or the operation's typed
   idempotency conflict for other mutations.
8. Existing revision rows are never updated or deleted.

## GoalContractV1

Typed structured content contained by a task revision.

```json
{
  "schema_version": 1,
  "goal": "Required primary goal",
  "included_scope": ["ordered"],
  "excluded_scope": [],
  "acceptance_criteria": ["ordered"],
  "constraints": ["ordered"]
}
```

### Normalization

For the goal and each list element, in this exact order:

1. convert `\r\n` to `\n`;
2. convert remaining `\r` to `\n`;
3. trim surrounding Unicode whitespace;
4. preserve internal whitespace, case, punctuation, and line breaks.

List order is preserved. Lists themselves may be empty. The primary goal and every
supplied list element must be non-empty after normalization.

### Canonical serialization and fingerprint

- Serialize a typed struct with fixed field declaration order shown above.
- Use compact UTF-8 JSON with standard JSON string escaping and no insignificant JSON
  whitespace.
- Include `schema_version` in the serialized and hashed bytes.
- Fingerprint is lowercase hexadecimal `BLAKE3(canonical_utf8_bytes)`.
- Deserialization validates the schema version and recomputed fingerprint.
- Errors expose stable violation enums such as `missing_goal`,
  `empty_goal`, `empty_list_item`, `unsupported_schema_version`, or
  `fingerprint_mismatch`; they never echo user content.

## Session binding dimensions

Feature 001 lifecycle remains:

```text
active | recovering | stopped | interrupted
```

Feature 002 adds independent binding mode:

```text
local_unbound
project_bound(project_id, task_revision_id)
```

Migration sets every existing session row to `local_unbound`. Lifecycle state,
timestamps, snapshots, leases, token hashes, and all events remain unchanged.

## SessionBinding

Immutable projection created once for one session.

| Field | Type | Rules |
|---|---|---|
| `session_id` | existing SessionId | Primary key; existing session |
| `project_id` | ProjectId | Active project at bind/start time |
| `task_revision_id` | TaskRevisionId | Revision of a task owned by project |
| `bound_at` | Timestamp | Immutable |
| `binding_event_seq` | integer | Unique global seq of `session.bound` |

Binding validation:

1. session exists;
2. current `sessions.binding_mode` is `local_unbound`, or existing binding is an
   identical retry;
3. project exists and is active;
4. session repository has an association to the project;
5. session worktree belongs to that same repository;
6. task revision exists and its task belongs to project.

Success appends `session.bound`, inserts this row, and updates only
`sessions.binding_mode`. Identical retry returns the same row. Any changed project or
revision returns `SESSION_BINDING_CONFLICT`. No unbind/rebind row transition exists.

## EventAggregateHead

Database-backed per-aggregate sequence allocator.

| Field | Type | Rules |
|---|---|---|
| `aggregate_type` | enum text | repository, worktree, session, project, task |
| `aggregate_id` | UUID text | Real aggregate identity |
| `last_seq` | non-negative integer | Incremented atomically before append |

Primary key is `(aggregate_type, aggregate_id)`. It is sequencing metadata derived
from event rows, not user project truth. Replay into a fresh database can reconstruct it
as the maximum `aggregate_seq` per aggregate.

## Events extension

Existing fields remain unchanged. Migration adds:

| Field | Type | Legacy rows | Post-migration rows |
|---|---|---|---|
| `aggregate_type` | enum text? | null | required |
| `aggregate_id` | UUID text? | null | required |
| `aggregate_seq` | positive integer? | null | required |

Constraints:

- all three aggregate fields are null together or non-null together;
- unique `(aggregate_type, aggregate_id, aggregate_seq)` when non-null;
- the post-migration event trigger rejects missing aggregate fields for every new event;
- existing update/delete denial triggers remain;
- global `seq` remains the authoritative complete replay order;
- `idempotency_key` remains globally unique.

Aggregate assignment:

| Event | Aggregate |
|---|---|
| `project.created`, `project.updated` | project |
| `project.repository_associated` | repository |
| `task.created`, `task.revision_created` | task |
| `session.bound` | session |
| Existing repository registration | repository |
| Existing worktree/snapshot/repository-state events | worktree |
| Existing session lifecycle events | session |

Stored legacy events remain null in aggregate columns. Replay derives their scope in
memory from existing real foreign keys, never by writing fake IDs.

## SQLite projection tables and indexes

### `projects`

Primary key `id`. Indexes: `(status, id)`, `(name, id)`.

### `project_repository_associations`

Primary key `id`. Unique `repository_id`; unique
`(project_id, repository_id)`; index `(project_id, repository_id)`.

### `tasks`

Primary key `id`. Indexes `(project_id, id)`, `(project_id, title, id)`.

### `task_revisions`

Primary key `id`. Unique `(task_id, revision_number)`; unique
`idempotency_key`; indexes `(task_id, revision_number DESC)`,
`parent_revision_id`.

### `session_bindings`

Primary key `session_id`. Unique `binding_event_seq`; indexes `project_id`,
`task_revision_id`.

### Existing `sessions`

Add `binding_mode TEXT NOT NULL DEFAULT 'local_unbound'` with closed check and index
`(binding_mode, state)`. Existing unique live-session index and every Feature 001
column remain unchanged.

### Existing `events`

Add aggregate columns, partial unique aggregate sequence index, and lookup index
`(aggregate_type, aggregate_id, aggregate_seq)`. Existing repository/worktree/session
indexes remain.

The design DDL is in
[contracts/migrations/0002_project_task_binding.sql](contracts/migrations/0002_project_task_binding.sql).

## Replay projections

Replay processes the complete ledger in ascending global `seq` and builds:

- `projects`: create then full-state updates;
- `project_repository_associations`: one immutable insert;
- `tasks`: create then advance latest revision;
- `task_revisions`: immutable inserts;
- `session_bindings`: initialize every `session.started` as local-unbound, then apply
  at most one `session.bound`;
- existing Feature 001 session lifecycle projection through its current replay path.

Replay equality compares stable persisted fields. Resume-token hashes and live lease
clocks remain authentication/runtime fields rather than event-sourced history, matching
Feature 001; migration preservation of those fields is verified separately.

## Deletion and foreign keys

All foreign keys use restrictive behavior; no cascade is defined. User-facing hard
delete operations do not exist. Storage tests prove immutable revision, association, and
binding rows cannot be changed through sanctioned DAOs. Append-only events remain
protected by existing SQLite triggers.
