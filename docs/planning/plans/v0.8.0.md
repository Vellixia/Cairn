# Cairn v0.8.0 — Implementation Plan

> **Goal:** Scoped memory (Session / Project / Global), SurrealDB replacing HelixDB+MinIO,
> auto project detection, LLM intelligence layer, RAG document store, session TTL + cron —
> plus an **autonomy layer** (S8–S9) so Cairn runs unattended day-to-day: memories auto-promote
> with undo instead of sitting in an approval queue, safe drift auto-approves while dangerous
> drift still waits for a human, workspace docs auto-sync into RAG, and the system self-heals
> (offline spool, cron watchdog, LLM budget cap) without losing any existing functionality.

---

## How to Read This Plan

Each task has:
- **What:** what you are building
- **Why:** why it is needed
- **Files:** exact files to create or edit
- **Changes:** exact struct fields, function signatures, and code to write
- **Done when:** specific verifiable condition

Work the sprints in order (S1 → S9). Each sprint produces a working, passing build.
Never start S2 before S1 compiles. Run `cargo check --workspace` after every sprint.

S1–S7 build the core migration (SurrealDB, scopes, auto project detection, session TTL + cron,
LLM intelligence, RAG, quality/dashboard). **S8–S9 are the autonomy layer**: they remove the
standing daily chores (approval queues, manual drift triage, manual doc ingestion) by replacing
them with act-by-default behavior, an observable log, cheap undo, and self-correcting safety
nets — the goal being Cairn needs far less day-to-day human interaction while every existing
function still works exactly as before for anyone who sets the relevant `*_AUTOPILOT=off` /
`CAIRN_SELFTUNE=off` env var.

### S1.4 — Create `cairn-store/src/surreal.rs`

**File:** `crates/cairn-store/src/surreal.rs` (new file)

This implements the `StoreBackend` trait (same interface as the old helix backend).
The `StoreBackend` trait is defined in `crates/cairn-store/src/lib.rs` — do not change it.

**Cargo.toml for `cairn-store`** — add to `[dependencies]`:
```toml
surrealdb = { workspace = true }
```

**Key functions to implement:**
```rust
use surrealdb::{Surreal, engine::any::Any};
use crate::StoreBackend;

pub struct SurrealStore {
    db: Surreal<Any>,
}

impl SurrealStore {
    /// Connect and apply schema. Called once at server startup.
    pub async fn open(url: &str, user: &str, pass: &str) -> Result<Self> {
        let db = surrealdb::engine::any::connect(url).await?;
        db.signin(surrealdb::opt::auth::Root { username: user, password: pass }).await?;
        db.use_ns("cairn").use_db("main").await?;
        db.query(include_str!("schema.surql")).await?.check()?;
        Ok(Self { db })
    }
}

impl StoreBackend for SurrealStore {
    // Implement each method by translating the old HelixDB calls to SurrealQL.
    // The trait method signatures do NOT change — only the implementation body changes.
}
```

For each `StoreBackend` method, map the HelixDB operation to SurrealDB:
- `store(memory)` → `db.create("memory").content(record).await?`
- `get(id)` → `db.select(("memory", id)).await?`
- `update(id, patch)` → `db.update(("memory", id)).merge(patch).await?`
- `delete(id)` → `db.delete(("memory", id)).await?`
- `list(filter)` → `db.query("SELECT * FROM memory WHERE ...").await?`
- `search_bm25(q)` → `db.query("SELECT *, search::score(1) AS score FROM memory WHERE content @1@ $q ORDER BY score DESC LIMIT $n").bind(("q", q)).await?`
- `search_vector(embedding, n)` → `db.query("SELECT *, vector::similarity::cosine(embedding, $vec) AS score FROM memory ORDER BY score DESC LIMIT $n").bind(("vec", embedding)).await?`

**HNSW index:** Create it after the first embedding is stored (we need the dimension):
```rust
pub async fn ensure_hnsw_index(&self, dim: usize) -> Result<()> {
    self.db.query(format!(
        "DEFINE INDEX IF NOT EXISTS idx_embedding ON memory \
         FIELDS embedding HNSW DIMENSION {dim} DIST COSINE TYPE F32;"
    )).await?.check()?;
    // Same for chunk table
    self.db.query(format!(
        "DEFINE INDEX IF NOT EXISTS idx_chunk_emb ON chunk \
         FIELDS embedding HNSW DIMENSION {dim} DIST COSINE TYPE F32;"
    )).await?.check()?;
    Ok(())
}
```

```rust
fn check_database(cfg: &cairn_core::Config) -> Check {
    // Attempt a simple HTTP/WS ping to CAIRN_DB_URL.
    // In remote mode (CAIRN_SERVER set) → skip (server handles DB).
    if std::env::var_os("CAIRN_SERVER").is_some() {

        .replace("wss://", "https://");

        },
        Err(e) => Check {
            name: "database", ok: false,
            detail: format!("{} (unreachable: {e})", cfg.db_url),
        },
    }
}
```

## Sprint 2 — Scope Model

**Goal:** Every memory has a `scope_type` + `scope_id`. Recall always returns
Global + current Project + current Session memories, weighted by scope.

**Corrected against reality (2026-07-02, after Sprint 1 shipped):** the original draft of
this sprint assumed `cairn-memory` builds raw SurrealQL strings against the store and that
`crates/cairn-api/src/middleware/` already existed. Neither is true. `cairn-memory` only has
`cairn-store`'s synchronous, backend-agnostic `Store` API (`all_memories()`,
`semantic_recall()`, etc.) - no SQL access, by design, so the storage engine can keep
changing (as it just did) without touching memory/ranking logic. `recall_for_org()`
(`crates/cairn-memory/src/lib.rs:193`) already does exactly this shape of filtering for
`org_id`: pull `all_memories()`, filter in Rust, then run BM25 + `semantic_recall` + graph
proximity through an inline RRF-weighted fusion (lines 264-328, no separate `rrf_fuse()`
function). There is no `middleware/` directory in `cairn-api` at all - `security_headers`
(a flat module, wired via `.layer(middleware::from_fn(security_headers::security_headers))`
in `lib.rs`) is the closest existing analog and the pattern to copy. There is also no
`access_log` table or concept anywhere yet (only a per-memory `access_count` counter), and
`OrgId` itself isn't wired from HTTP into `MemoryEngine` today (call sites hardcode
`OrgId::default()`) - Sprint 2 is genuinely the first per-request scope context Cairn has.
The plan below reflects the corrected design; the sections after it were rewritten from
scratch, not lightly edited.

### S2.2 — `ScopeCtx` extractor in `cairn-api` (not `Config` — scope is per-request)

**File:** `crates/cairn-api/src/scope.rs` (new, flat module — mirrors `security_headers`,
not a `middleware/` subdirectory)

Project/session context is inherently per-HTTP-request, not a static server setting, so it
does not belong on `Config` (the original draft's S2.2 was wrong on this point too).

```rust
#[derive(Clone, Debug, Default)]
pub struct ScopeCtx {
    pub project_id: Option<String>,
    pub session_id: Option<String>,
}

pub async fn scope_middleware(headers: HeaderMap, mut req: Request, next: Next) -> Response {
    let project_id = headers.get("X-Cairn-Project").and_then(|v| v.to_str().ok()).map(str::to_string);
    let session_id = headers.get("X-Cairn-Session").and_then(|v| v.to_str().ok()).map(str::to_string);
    req.extensions_mut().insert(ScopeCtx { project_id, session_id });
    next.run(req).await
}
```

Wire it in `crates/cairn-api/src/lib.rs` next to the existing `security_headers` layer.
Handlers that need it take `Extension(ctx): Extension<ScopeCtx>` as an extractor argument
(standard axum `Extension` pattern - no new plumbing needed beyond the layer itself).

```rust
.filter(|m| {
```

### S2.5 — Batched `access_log` writes

**File:** `crates/cairn-store/src/schema.surql` (new table), `crates/cairn-store/src/db.rs`
(new `StoreBackend` trait method with a default no-op impl - same pattern as

```rust
// db.rs trait addition (default no-op so memory_backend.rs doesn't need new logic):
fn record_access_batch(&self, entries: &[(String, Option<String>, Option<String>)]) -> Result<()> {
    let _ = entries;
    Ok(())
}
```

`surreal.rs` implements it as one multi-row `INSERT INTO access_log $rows` - a busy agent
recalling every turn can generate thousands of rows/hour, and one round-trip per recall
call (not one per returned memory) is what actually matters here; a background
`tokio::spawn` is unnecessary complexity for a single batched write and was dropped from
the original draft. The S4 `access-log-prune` cron job (`CAIRN_ACCESS_LOG_RETENTION_DAYS`,
default 90) keeps the table bounded regardless of write volume.

```rust
fn detect_project() -> (Option<String>, String) {
    // Priority 1: CAIRN_PROJECT env var (explicit override)
    if let Ok(name) = std::env::var("CAIRN_PROJECT") {
        if !name.trim().is_empty() {
            let id = project_hash(&name);
            return (Some(id), name);
        }
    }

    // Priority 2: git rev-parse --show-toplevel
    if let Ok(out) = std::process::Command::new("git")
        .args(["rev-parse", "--show-toplevel"])
        .output()
    {
        if out.status.success() {
            let path = String::from_utf8_lossy(&out.stdout).trim().to_string();
            let canonical = std::fs::canonicalize(&path)
                .unwrap_or_else(|_| std::path::PathBuf::from(&path));
            let name = canonical.file_name()
                .map(|n| n.to_string_lossy().into_owned())
                .unwrap_or_else(|| "unknown".to_string());
            let id = project_hash(&canonical.to_string_lossy());
            return (Some(id), name);
        }
    }

    // Priority 3: $PWD basename
    if let Ok(cwd) = std::env::current_dir() {
        let name = cwd.file_name()
            .map(|n| n.to_string_lossy().into_owned())
            .unwrap_or_else(|| "unknown".to_string());
        let id = project_hash(&cwd.to_string_lossy());
        return (Some(id), name);
    }

    (None, "unknown".to_string())
}

fn project_hash(path: &str) -> String {
    use sha2::{Sha256, Digest};
    let hash = Sha256::digest(path.as_bytes());
    hex::encode(&hash[..8])  // 16 hex chars = 64 bits, collision-resistant enough
}

fn current_dir_str() -> String {
    std::env::current_dir()
        .ok()
        .map(|p| p.to_string_lossy().into_owned())
        .unwrap_or_default()
}
```

```rust
pub async fn consolidate_stale_sessions(
    store: &dyn StoreBackend,
    ttl_days: u32,
    scope_default: &str,
    llm: Option<&LlmClient>,
) -> Result<usize> {
    if ttl_days == 0 { return Ok(0); }

    let cutoff = Utc::now() - Duration::days(ttl_days as i64);
    let stale = store.query(
        "SELECT * FROM session WHERE ended_at IS NULL AND started_at < $cutoff"
    ).bind(("cutoff", cutoff)).await?;

    let mut count = 0;
    for session in stale {
        // Step 1: working → episodic
        store.query(
            "UPDATE memory SET tier = 'episodic' \
             WHERE scope_type = 'session' AND scope_id = $sid AND tier = 'working'"
        ).bind(("sid", &session.id)).await?;

        // Step 2: session scope → project or global
        let target_scope = if session.project_hash.is_some() { "project" } else { scope_default };
        let target_id    = session.project_hash.as_deref();
        store.query(
            "UPDATE memory SET scope_type = $ts, scope_id = $tid \
             WHERE scope_type = 'session' AND scope_id = $sid"
        )
        .bind(("ts", target_scope))
        .bind(("tid", target_id))
        .bind(("sid", &session.id))
        .await?;

        // Step 3: LLM session summary (only if LLM enabled)
        if let Some(llm) = llm {
            let mems: Vec<Memory> = store.query(
                "SELECT content FROM memory WHERE scope_id = $sid"
            ).bind(("sid", &session.id)).await?;
            if !mems.is_empty() {
                let summary = llm.summarize_session(&mems).await?;
                store.create_memory(NewMemory {
                    content:    summary,
                    kind:       "note".into(),
                    tier:       "episodic".into(),
                    scope_type: ScopeType::Project,
                    scope_id:   session.project_hash.clone(),
                    importance: 0.6,
                    ..Default::default()
                }).await?;
            }
        }

        // Step 4: mark ended
        store.query(
            "UPDATE session SET ended_at = $now WHERE id = $sid"
        ).bind(("now", Utc::now())).bind(("sid", &session.id)).await?;

        count += 1;
    }
    Ok(count)
}
```

```rust
pub async fn detect_contradictions(
    store: &dyn StoreBackend,
    llm: &LlmClient,
) -> Result<usize> {
    let new_mems: Vec<Memory> = store.query(
        "SELECT * FROM memory WHERE created_at > $since AND suspicious = false LIMIT 50"
    ).bind(("since", Utc::now() - Duration::hours(24))).await?;

    let mut flagged = 0;
    for mem in new_mems {
        // Find top-10 semantically similar memories
        let similar: Vec<Memory> = store.search_vector(&mem.embedding, 10).await?;
        if similar.is_empty() { continue; }

        let candidates_str: Vec<String> = similar.iter().map(|m| {
            format!("ID: {} — {}", m.id, m.content)
        }).collect();

        let prompt = format!(
            "Does this statement contradict any of the candidates? \
             Statement: {}\n\nCandidates:\n{}\n\n\
             Return JSON: [{{\"id\": \"...\", \"contradicts\": true/false}}] for each candidate.",
            mem.content,
            candidates_str.join("\n")
        );

        if let Ok(json_str) = llm.complete(&prompt).await {
            if let Ok(results) = serde_json::from_str::<Vec<ContradictionResult>>(&json_str) {
                for r in results {
                    if r.contradicts {
                        // RELATE memory:mem_id ->contradicts-> memory:r.id
                        store.query(
                            "RELATE memory:$a ->contradicts-> memory:$b"
                        ).bind(("a", &mem.id)).bind(("b", &r.id)).await?;
                        store.query("UPDATE memory SET suspicious = true WHERE id = $id")
                            .bind(("id", &mem.id)).await?;
                        flagged += 1;
                    }
                }
            }
        }
    }
    Ok(flagged)
}
```

### S5.5 — Smart promotion scoring pipeline

**File:** `crates/cairn-memory/src/llm_intelligence.rs`

This runs inside the `llm-intelligence` cron job. Scores each `Project`-scoped
memory at `Semantic` or `Procedural` tier.

```rust
pub async fn score_promotion_candidates(
    store: &dyn StoreBackend,
    llm: &LlmClient,
    global_centroid: &[f32],  // precomputed by global-centroid job
) -> Result<Vec<PromotionCandidate>> {
    let candidates: Vec<Memory> = store.query(
        "SELECT * FROM memory \
         WHERE scope_type = 'project' \
         AND tier IN ['semantic', 'procedural'] \
         AND promo_locked = false \
         AND promo_score < 0.90"
    ).await?;

    let mut scored = Vec::new();
    for mem in candidates {
        // Step 1: Hard blocklist
        if contains_sensitive(&mem.content) { continue; }

        // Step 2: Kind prior
        let kind_prior = kind_prior_score(&mem.kind);
        if kind_prior == 0.0 { continue; }

        // Step 3: Cross-project signal
        let cross_count: i64 = store.query(
            "SELECT count() FROM access_log \
             WHERE memory_id = $id AND project_id != $pid \
             AND ts > $cutoff GROUP ALL"
        ).bind(("id", &mem.id))
         .bind(("pid", mem.scope_id.as_deref().unwrap_or("")))
         .bind(("cutoff", Utc::now() - Duration::days(30)))
         .await?;
        let cross_score = (cross_count as f32 / 3.0).min(1.0);

        // Step 4: Embedding universality
        let univ_score = cosine_similarity(&mem.embedding, global_centroid);

        // Step 5: Fast score
        let fast_score = kind_prior * 0.2 + cross_score * 0.5 + univ_score * 0.3;

        // Step 6: LLM judgment only in borderline range
        let final_score = if fast_score >= 0.40 && fast_score <= 0.85 {
            let prompt = format!(
                "Is this knowledge project-specific or universally applicable to any project? \
                 Score from 0.0 (project-only) to 1.0 (universal). Reply with just the number.\n\n{}",
                mem.content
            );
            if let Ok(s) = llm.complete(&prompt).await {
                let llm_score = s.trim().parse::<f32>().unwrap_or(fast_score);
                fast_score * 0.5 + llm_score * 0.5
            } else { fast_score }
        } else { fast_score };

        // Step 7: Update DB score
        store.query("UPDATE memory SET promo_score = $s WHERE id = $id")
            .bind(("s", final_score)).bind(("id", &mem.id)).await?;

        // Auto-promote if very high
        if final_score > 0.90 {
            store.query(
                "UPDATE memory SET scope_type = 'global', scope_id = NONE WHERE id = $id"
            ).bind(("id", &mem.id)).await?;
        }

        scored.push(PromotionCandidate { memory: mem, score: final_score });
    }
    Ok(scored)
}

fn contains_sensitive(text: &str) -> bool {
    // Regex for IP address, email, absolute file path
    static RE: Lazy<Regex> = Lazy::new(|| Regex::new(
        r"(\d{1,3}\.){3}\d{1,3}|[\w.]+@[\w.]+\.\w+|(/home|/Users|C:\\)[^\s]+"
    ).unwrap());
    RE.is_match(text)
}

fn kind_prior_score(kind: &str) -> f32 {
    match kind {
        "procedural" => 1.0,
        "fact"       => 0.9,
        "gotcha"     => 0.7,
        "decision"   => 0.6,
        "note"       => 0.3,
        "task"       => 0.0,
        "preference" => 0.0,
        _            => 0.3,
    }
}
```

```rust
pub async fn recompute_global_centroid(store: &dyn StoreBackend) -> Result<Vec<f32>> {
    let embeddings: Vec<Vec<f32>> = store.query(
        "SELECT embedding FROM memory WHERE scope_type = 'global' AND array::len(embedding) > 0"
    ).await?;
    if embeddings.is_empty() { return Ok(vec![]); }
    let dim = embeddings[0].len();
    let mut centroid = vec![0.0f32; dim];
    for emb in &embeddings {
        for (i, v) in emb.iter().enumerate() { centroid[i] += v; }
    }
    let n = embeddings.len() as f32;
    centroid.iter_mut().for_each(|v| *v /= n);
    Ok(centroid)
}
```

```rust
    store: Arc<dyn StoreBackend>,

impl DocumentEngine {
    pub async fn ingest(
        &self,
        source: DocumentSource,
        project_id: Option<String>,
    ) -> Result<Document> {
        let (raw_text, title) = match &source {
            DocumentSource::File(p) => (std::fs::read_to_string(p)?, p.display().to_string()),
            DocumentSource::Text { content, title } => (content.clone(), title.clone()),
            DocumentSource::Url(_) => todo!("URL fetch in a later patch"),
        };

        // Create document record
        let doc_id = uuid::Uuid::new_v4().to_string();
        let doc = Document { id: doc_id.clone(), project_id: project_id.clone(),
            source: title.clone(), title: title.clone(), created_at: Utc::now() };
        self.store.query("INSERT INTO document $d").bind(("d", &doc)).await?;

        // Chunk with text-splitter (512 tokens, 64 overlap, markdown-aware)
        use text_splitter::{TextSplitter, ChunkConfig};
        let splitter = TextSplitter::new(ChunkConfig::new(512).with_overlap(64)?);
        let chunks: Vec<&str> = splitter.chunks(&raw_text).collect();

        let mut char_offset = 0;
        for chunk_text in chunks {
            let embedding = self.embed.embed(chunk_text).await?;
            let tok_count = chunk_text.split_whitespace().count(); // rough estimate
            self.store.query(
                "INSERT INTO chunk { document_id: $did, text: $t, embedding: $e, \
                 token_count: $tc, char_offset: $co }"
            )
            .bind(("did", &doc_id))
            .bind(("t", chunk_text))
            .bind(("e", embedding))
            .bind(("tc", tok_count))
            .bind(("co", char_offset))
            .await?;
            char_offset += chunk_text.len();
        }

        Ok(doc)
    }

    pub async fn search(
        &self,
        query: &str,
        project_id: Option<&str>,
        limit: usize,
    ) -> Result<Vec<ScoredChunk>> {
        let qvec = self.embed.embed(query).await?;
        let scope_filter = match project_id {
            Some(pid) => format!("AND document.project_id IN ['{pid}', NULL]"),

             vector::similarity::cosine(chunk.embedding, $vec) AS score \
             FROM chunk \
             JOIN document ON chunk.document_id = document.id \
             WHERE 1=1 {scope_filter} \
             ORDER BY score DESC LIMIT {limit}"
        )).bind(("vec", qvec)).await?;
        Ok(results)
    }
```

### S6.3b — Auto-sync workspace docs (removes the "remember to `cairn documents ingest`" chore)

**What:** a `sync_workspace` pass that keeps RAG current without a human running the ingest CLI.
**Why:** manual ingestion means docs silently go stale — nobody remembers to re-run it after
editing a README. Auto-sync makes "workspace docs are searchable" a property of the system, not
a task on someone's list. `cairn documents ingest` stays for URLs and one-off files.

```rust
impl DocumentEngine {
    /// Walk `root` for files matching `globs` (default `**/*.md`), gitignore-aware, capped at
    /// `CAIRN_DOCS_SYNC_MAX_FILES` (default 500) and `CAIRN_DOCS_SYNC_MAX_FILE_KB` (default 512)
    /// per file. Ingests new/changed files by content hash (skip unchanged — same content-hash
    /// idea `cairn-context` already uses for cache invalidation), and prunes chunks whose source
    /// file no longer exists. Idempotent: running twice with no filesystem changes is a no-op.
    pub async fn sync_workspace(
        &self,
        root: &Path,
        globs: &[&str],
        project_id: Option<&str>,
    ) -> Result<SyncReport> {
        let files = walk_matching(root, globs)?;              // respects .gitignore
        let mut ingested = 0;
        let mut skipped = 0;
        for path in &files {
            let content = std::fs::read_to_string(path)?;
            let hash = ContentHash::of_str(&content);
            let existing = self.store.query(
                "SELECT id, content_hash FROM document WHERE source = $src"
            ).bind(("src", path.display().to_string())).await?;
            match existing.first() {
                Some(d) if d.content_hash == hash.as_str() => { skipped += 1; continue; }
                Some(d) => { self.delete(&d.id).await?; } // changed — re-chunk below
                None => {}
            }
            self.ingest(
                DocumentSource::Text { content, title: path.display().to_string() },
                project_id.map(str::to_string),
            ).await?;
            ingested += 1;
        }
        // Prune: any document whose source file is gone from `files`.
        let pruned = self.prune_missing_sources(root, &files).await?;
        Ok(SyncReport { ingested, skipped, pruned })
    }
}
```

**Triggered from two places** (both call the same function; no duplicated logic):
- `SessionStart` hook — debounced (skip if the last sync for this workspace was < 5 minutes ago,
  tracked in a small local marker file) so opening many sessions in a row doesn't re-walk the
  tree every time.
- Nightly `docs-sync` cron job (added alongside the other jobs in S4.3), for workspaces that sit
  open for days without a fresh `SessionStart`.

### S7.1 — MMR in base recall (not just hybrid_search)

**File:** `crates/cairn-memory/src/lib.rs`

Currently MMR (Maximal Marginal Relevance) is only in `hybrid_search()`. Apply it to the
base `recall()` function too so all recall paths benefit from diversity:

```rust
fn mmr_rerank(results: Vec<ScoredMemory>, lambda: f32, k: usize) -> Vec<ScoredMemory> {
    // Standard MMR: iteratively pick the item that maximizes
    //   lambda * sim(query, item) - (1-lambda) * max_sim(item, already_selected)
    // lambda=0.7 = relevance-heavy; 0.3 = diversity-heavy
    // k = how many to return
    let mut selected: Vec<ScoredMemory> = Vec::new();
    let mut remaining = results;
    while selected.len() < k && !remaining.is_empty() {
        let best = remaining.iter().enumerate().max_by(|(_, a), (_, b)| {
            let a_rel = a.score;
            let a_div = selected.iter()
                .map(|s| cosine_similarity(&a.embedding, &s.embedding))
                .fold(0.0f32, f32::max);
            let a_mmr = lambda * a_rel - (1.0 - lambda) * a_div;
            let b_rel = b.score;
            let b_div = selected.iter()
                .map(|s| cosine_similarity(&b.embedding, &s.embedding))
                .fold(0.0f32, f32::max);
            let b_mmr = lambda * b_rel - (1.0 - lambda) * b_div;
            a_mmr.partial_cmp(&b_mmr).unwrap_or(Equal)
        }).map(|(i, _)| i);
        if let Some(i) = best {
            selected.push(remaining.remove(i));
        }
    }
    selected
}
```

Apply after RRF fusion with `lambda=0.7`, `k=limit`.

### S8.2 — Nightly auto-demote (the promotion safety net)

```rust
pub async fn demote_stale_globals(store: &dyn StoreBackend, idle_days: u32) -> Result<usize> {
    // Candidate: Global, auto-promoted (has a promotion_log row), not human-pinned/promoted,
    // and EITHER marked suspicious by contradiction detection (S5.3/S8.3) OR zero cross-project
    // access_log hits in `idle_days`.
    let candidates = store.query(
        "SELECT * FROM memory WHERE scope_type = 'global' AND pinned = false \
         AND id IN (SELECT memory_id FROM promotion_log WHERE reason = 'auto-threshold') \
         AND (suspicious = true OR id NOTIN (\
             SELECT memory_id FROM access_log WHERE ts > $cutoff))"
    ).bind(("cutoff", Utc::now() - Duration::days(idle_days as i64))).await?;

    let mut demoted = 0;
    for mem in candidates {
        store.query(
            "UPDATE memory SET scope_type = 'project', scope_id = $pid WHERE id = $id"
        ).bind(("pid", mem.original_project_id())).bind(("id", &mem.id)).await?;
        store.query(
            "CREATE promotion_log SET memory_id=$id, old_scope='global', score=0.0, \
             reason='auto-demote', ts=$t"
        ).bind(("id", &mem.id)).bind(("t", Utc::now())).await?;
        demoted += 1;
    }
    Ok(demoted)
}
```

### S8.4 — Drift autopilot (removes the reject/approve chore)

**What:** safe drift auto-approves; dangerous drift always waits for a human.
**Why:** today every single drift event — including a `risk=ok` verify on a comment typo —

**File:** `crates/cairn-guard/src/lib.rs` — add a policy function:

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum DriftAutopilot { Off, Safe, All }

impl DriftAutopilot {
    pub fn from_env() -> Self {
        match std::env::var("CAIRN_DRIFT_AUTOPILOT").as_deref() {
            Ok("off") => Self::Off,
            Ok("all") => Self::All,
            _ => Self::Safe, // default
        }
    }
}

/// Decide whether a drift event auto-approves. `danger` NEVER auto-approves, regardless of mode.
pub fn autopilot_decision(mode: DriftAutopilot, risk: RiskLevel, path: &str, safe_globs: &[&str]) -> Option<&'static str> {
    if risk == RiskLevel::Danger { return None; } // always hold for a human
    match mode {
        DriftAutopilot::Off => None,
        DriftAutopilot::All => Some("autopilot:all"),
        DriftAutopilot::Safe => match risk {
            RiskLevel::Ok => Some("autopilot:ok"),
            RiskLevel::Warn if glob_match_any(path, safe_globs) => Some("autopilot:safe-path"),
            _ => None,
        },
    }
}
```

**Why:** the anchor is what re-orients a drifting long task, but today it only exists if someone
remembers to call `anchor` — the exact chore this sprint's principle targets.

**File:** `crates/cairn-proactive/src/lib.rs` (or the `UserPromptSubmit` path in
`crates/cairn-client/src/hook.rs`, whichever already owns proactive triggers): on
`UserPromptSubmit`, if `GET /api/guard/anchor` returns empty for this session, derive a one-line
anchor from the prompt (LLM-summarized when `CAIRN_LLM_ENABLED=true`; otherwise the first
sentence, truncated) and `POST` it. Re-derive only if the current prompt's embedding is far from
the existing anchor's embedding (topic shift), not on every turn.

**Config:** `CAIRN_AUTO_ANCHOR=true` (default) — added to S1.9's env block.

**Done when:** a fresh session with no manual `anchor` call still shows a task anchor in
`GET /api/guard/anchor` after the first prompt; an existing manually-set anchor is left alone
unless the topic clearly shifts.

### S8.6 — SessionStart digest + weekly rollup (replaces "go check the dashboard")

**What:** summarize what autopilot did while the human was away, delivered inside the agent.
**Why:** if promotion/drift/doc-sync all run unattended (S8.1–S8.4, S6.3b), the human's only way
to notice a mistake is visiting the dashboard — reintroducing the very habit this sprint removes.
Put the summary where they already are: the next session's context.

**File:** `crates/cairn-client/src/hook.rs` — in the `SessionStart` handler, alongside the
existing wakeup/anchor context block, add one more `GET` for a digest:
```
GET /api/memory/autopilot-digest?since=<last_session_end>
→ { "promoted": N, "demoted": N, "drift_auto_approved": N, "contradictions_resolved": N,
    "docs_synced": N }
```
Render as one line in the injected context: `"Since you were away: N memories auto-promoted,
N drift events auto-approved, N docs re-synced — see /memory?tab=log for detail."` Omit the line

**File:** `crates/cairn-api/src/cron.rs` — a `weekly-rollup` job composes the same counts into one
audit-log entry per week (and a push notification via the existing `cairn-api` push subscription
store, if the user has one registered).

**Done when:** starting a new session after a night of autopilot activity shows a one-line
summary with no dashboard visit required; a quiet night (zero autopilot actions) shows nothing.

## Sprint 9 — Resilience & self-tuning (safe to leave unattended)

**Goal:** the autonomy added in S8 must not become a new failure mode when the network drops,
a cron job hangs, or the LLM bill runs away. This sprint makes unattended operation *safe*, not
just *possible*.

---

### S9.1 — Offline spool for hooks (never lose a memory when the server is down)

**What:** if the server is unreachable, hooks queue the payload locally instead of dropping it.
**Why:** today a `remember`/`consolidate` call from a hook that hits a down server is just lost
(best-effort, per `hook.rs`'s "must never break the agent" design) — acceptable for resilience,
not for an unattended system where "the server restarted overnight" shouldn't cost real memories.

**File:** `crates/cairn-client/src/hook.rs` — wrap the existing remote-mode HTTP calls: on
connection failure, append `{event, payload, ts}` as one line to `~/.cairn/spool.jsonl` instead
of silently dropping it. On the next successful `SessionStart` (i.e. the server answered), read
the spool, replay each entry against the same endpoints it originally targeted, and truncate the
file only after every entry is confirmed sent. Hooks still exit 0 either way — this changes what
happens to the payload, not the hook's failure contract with the agent.

**Done when:** stopping the server, running a hook that would call it, and confirming
`spool.jsonl` gained a line; starting the server and running `SessionStart` flushes the spool and
empties the file; the payloads land in the store exactly as if the server had been up throughout.

---

### S9.2 — Cron watchdog + single-flight (self-healing scheduler)

**What:** cron jobs record their own health and refuse to pile up if one run overruns.
**Why:** S4's cron jobs plus S8's new ones (`memory-demote`, `docs-sync`, `weekly-rollup`) mean
more scheduled work than before; without a watchdog, a stuck job (e.g. `docs-sync` on a huge
monorepo) could overlap with its own next tick indefinitely.

**File:** `crates/cairn-api/src/cron.rs` — extend `CronRunner` (S4.3) so each job records
`last_run_at`, `last_status`, `duration_ms` in-memory (or a small `cron_run` table), and wraps
every job body in a single-flight guard (an `AtomicBool` or a per-job `tokio::sync::Mutex::try_lock`)
that **skips** a tick if the previous run of the same job is still in flight, logging the skip
rather than queuing a pile-up.

**File:** `crates/cairn-api/src/handlers/cron.rs` — add `GET /api/cron/health`: per-job
`last_run_at`, `last_status`, and `stale: bool` (true when a job hasn't completed successfully
within `2×` its own schedule interval). S8.6's digest surfaces `stale: true` jobs so a human
notices a wedged scheduler without needing to check `/api/cron/jobs` proactively.

**Done when:** manually stalling one job (e.g. a debug `sleep`) and firing its schedule again
produces a logged skip, not a second concurrent run; `GET /api/cron/health` reflects the stall.

---

### S9.3 — LLM budget guard (autonomy that can't run away with cost)

**What:** a daily token budget for background LLM calls; jobs degrade gracefully instead of
overspending or hard-failing when it's exhausted.
**Why:** S5's LLM intelligence jobs and S8.1/S8.3's promotion/contradiction scoring all make LLM
calls in the background with no human approving each one — the same autonomy that removes a
chore also removes the natural cost checkpoint a human approval step used to provide.

**File:** `crates/cairn-memory/src/llm_consolidator.rs` (the existing OpenAI-compat client, S5's
reuse target) — add a `LlmBudget` tracker (in-memory counter reset daily, persisted to survive
restarts): before each background batch (concept extraction, contradiction detection, promotion
LLM-judgment step), check remaining budget. If exhausted, skip the LLM step and fall back to the
fast heuristic path that already exists for each of those flows (S5.5's `fast_score` without the
LLM blend, S5.2/S5.3 simply deferred to the next day) — never block, never silently keep spending.

**Config:** `CAIRN_LLM_DAILY_BUDGET=200000` (tokens/day; `0` = unlimited) — added to S1.9's env
block. Budget usage exposed at `GET /api/metrics` and included in S8.6's digest.

**Done when:** setting the budget artificially low and running the `llm-intelligence` cron job
shows it falling back to heuristics after the budget is spent, with a logged skip reason — not a
crash, not an unbounded bill.

---

### S9.4 — Self-tuning retrieval thresholds (gets smarter without a human tuning it)

**What:** a weekly job nudges promotion threshold, proactive-recall sensitivity, and MMR lambda
based on measured retrieval quality, instead of those constants staying hand-picked forever.
**Why:** the existing bounce tracker (`cairn-context/src/bounce_tracker.rs`) and followup-rate
metric (mentioned in `docs/planning/plans/v0.7.0.md`'s Sprint plan) already measure "did the agent
have to re-read/re-query after we gave it an answer" — that signal is presently only visible on
a dashboard chart. Feeding it back into the thresholds closes the loop automatically.

**File:** `crates/cairn-proactive/src/lib.rs` — new `tune` cron job (S4.3's job list): read the
past week's bounce rate and followup rate; if bounce rate is high (agent re-read files it was
given a compressed view of too aggressively), nudge context thresholds toward less compression;
if followup rate is high (recall isn't surfacing the right things), nudge `CAIRN_PROMOTE_THRESHOLD`
or MMR `lambda` within a **bounded range** (e.g. ±0.05 per week, hard floor/ceiling) — never a
large jump from one run. Every adjustment writes an audit-log entry with before/after values and
the signal that triggered it.

**Config:** `CAIRN_SELFTUNE=on` (default) `|off` — added to S1.9's env block. `off` pins all
thresholds at their configured/default values exactly as in S1–S7.

**Done when:** seeding a high bounce-rate signal and running the `tune` job moves a threshold
within its bounded range and logs the change; `CAIRN_SELFTUNE=off` leaves thresholds untouched
across repeated runs.

---

### S9.5 — Memory hygiene (keeps the store healthy unattended)

**What:** fold a few housekeeping passes into the existing decay cron so an unattended store
doesn't accumulate cruft indefinitely.
**Why:** S8's autonomy increases write volume (auto-promotion, auto-anchor, auto-drift-approval
all write more than a fully-manual system would) — the store needs the same "runs itself" property
the rest of this sprint gives the workflow.

**File:** `crates/cairn-memory/src/lib.rs` — extend the `memory-decay` cron job (S4's job table)
with three passes, each independently toggleable but on by default:
1. **Dedup sweep** — SHA-256 content-hash match among Working-tier memories (the same hash
   `remember`/`remember_for_org` already computes) that somehow diverged in tier; merge into one.
2. **Expired-session cleanup** — Session-scoped memories whose session TTL (S4.1) elapsed without
   ever consolidating (crash, killed process) get force-consolidated by the same path S4.2 uses.
3. **Per-project Working-tier cap** — `CAIRN_MAX_WORKING_PER_PROJECT` (default 500); oldest
   lowest-confidence rows beyond the cap are evicted from the *hot* tier — the full record stays
   retrievable via the blob store per Cairn's lossless-retention guarantee (this is a tier
   demotion for query performance, never a data deletion).

**Done when:** the three passes run as part of the existing weekly `memory-decay` job with no new
cron entry; each is individually disableable via env without touching the others.
